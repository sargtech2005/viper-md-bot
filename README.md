# 🐍 VIPER BOT MD — Web Platform

WhatsApp bot pairing platform. Users sign up, get free coins, create sessions, pair numbers — all from a dashboard.

---

## 🚀 Deploy on Render (step by step)

### 1. Push to GitHub
```bash
git init
git add .
git commit -m "VIPER BOT MD web platform"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

### 2. Create Render account
Go to [render.com](https://render.com) and sign up free.

### 3. Deploy with render.yaml (easiest)
1. Render Dashboard → **New** → **Blueprint**
2. Connect your GitHub repo
3. Render reads `render.yaml` and creates both the **Web Service** and **PostgreSQL** database automatically
4. Click **Apply**

### 4. Set your ADMIN_EMAIL
After deploy:
1. Render Dashboard → your web service → **Environment**
2. Add: `ADMIN_EMAIL` = `your@email.com`
3. Click **Save Changes** → service will redeploy

### 5. Done!
- Your site: `https://viper-bot-md.onrender.com`
- Register with your `ADMIN_EMAIL` → you'll automatically get admin access

## ⚠️ Render Free Tier — What You Need to Know

| Thing | Free Tier Behaviour | Impact |
|---|---|---|
| **Web service** | Spins down after 15 min of no traffic | First request after idle takes ~30s to wake up |
| **File system** | Ephemeral — wiped on every restart | ✅ Handled: credentials saved in Postgres |
| **PostgreSQL** | Free for 90 days, then expires | Back up your DB before day 90 |
| **Memory** | 512 MB | Can run ~5–10 bot sessions comfortably |

### Keep it awake (optional)
Use [UptimeRobot](https://uptimerobot.com) (free):
1. New monitor → HTTP(s)
2. URL: `https://your-app.onrender.com/health`
3. Interval: every 5 minutes

This prevents the 30s cold start for your users.

## ⚙️ Environment Variables

Set these in **Render Dashboard → Environment**:

| Variable | Description |
|---|---|
| `DATABASE_URL` | Auto-set by Render when you link the DB |
| `JWT_SECRET` | Auto-generated by Render |
| `ADMIN_EMAIL` | **Set this manually** — gets auto-admin on first login |
| `NODE_ENV` | `production` (set in render.yaml) |
| `PORT` | `3000` (set in render.yaml) |

All other settings (Paystack keys, coin prices, signup bonus, max sessions) are in the **Admin Panel** — no env vars needed, no restart required.

## 💳 Paystack Setup

1. Create account at [paystack.com](https://paystack.com)
2. Go to **Settings → Developer → API Keys**
3. Log into your site → Admin Panel → **Paystack Keys** tab
4. Paste **Public Key** + **Secret Key** → Save
5. Copy the **Webhook URL** shown → paste in Paystack Dashboard → API → Webhooks

## 🗃 About the Database

The `render.yaml` automatically creates a **Render PostgreSQL** database and links it to your web service via `DATABASE_URL`. The app creates all tables automatically on first boot — no manual SQL needed.

**⚠️ Free Postgres expires after 90 days.** Before expiry:
- Render Dashboard → your DB → **Backups** → Export
- Or run: `pg_dump $DATABASE_URL > backup.sql`

## 📱 How Pairing Works

1. User creates a session (costs 10 coins)
2. Enters WhatsApp number in full international format
3. Server spawns a Node.js bot process for that number
4. Bot requests a pairing code from WhatsApp via Baileys
5. Code streams to the browser in real-time via SSE
6. User enters code in WhatsApp → **Settings → Linked Devices → Link with phone number**
7. Bot connects → session credentials are saved to PostgreSQL
8. On any server restart, credentials are restored from DB — **no re-pairing needed**
